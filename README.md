Общий пайплайн для решения соревок с табличными данными (и регрессия и классификация)

### Описание проекта  
  
Этот репозиторий содержит экспериментальный ML‑проект для участия в соревнованиях Kaggle:  
  
- **Titanic — Machine Learning from Disaster** (задача бинарной классификации: выжил / не выжил)  
- **House Prices — Advanced Regression Techniques** (задача регрессии: предсказание цены дома)  
  
Вся логика обучения и инференса вынесена в класс `Solver`, конфигурация управляется через `config.yaml`, а предварительная обработка данных инкапсулирована в классах датасетов.  
  
### Структура проекта  
  
- `main.py` — точка входа. Загружает конфиг, создает `Solver` и запускает обучение / инференс.  
- `solver.py` — реализация класса `Solver`: обучение моделей, кросс‑валидация, сохранение чекпоинтов, инференс, ансамблирование / стакинг.  
- `dataset.py` 
- `utils.py` — вспомогательные функции (например, `set_seed` для фиксации детерминированности экспериментов).  
- `config.yaml` — единый конфигурационный файл с путями к данным, режимами работы, параметрами валидации и гиперпараметрами моделей.
  
Ожидается, что рядом с кодом будут каталоги с данными Kaggle:  
  
- `./titanic/train.csv`, `./titanic/test.csv`  
- `./pricing/train.csv`, `./pricing/test.csv`

### Конфигурация (`config.yaml`)  
  
Основные блоки конфига:  
  
- **`task`** — выбор задачи (может быть активна только одна):  
  - `titanic: True / False`  
  - `house_pricing: True / False`  
- **`path`** — пути к train / test CSV, директориям с моделями и чекпоинтами, а также к итоговым submission‑файлам.  
- **`mode`**:  
  - `is_train` — запуск обучения моделей.  
  - `is_inference` — запуск инференса и генерации submission.  
  - флаги отдельных моделей: `LinearRegression`, `Lasso`, `Ridge`, `ElasticNet`, `KNN`, `DecisionTree`, `RandomForest`, `CatBoost`, `XGB`, `LGBM`, `DNN`.  
- **`validation`** — параметры `StratifiedKFold` (для Titanic) и `KFold` (для House Prices).  
- **`inference.ensemble`**:  
  - для Titanic — софт‑вотинг по вероятностям;  
  - для House Prices — стакинг (`is_stacking`) или софт‑вотинг (`is_voting`).  
- **`params`** — гиперпараметры всех поддерживаемых моделей.  
  
Все ключевые режимы и модели включаются / выключаются редактированием `config.yaml`, без изменения кода.    
  
### Запуск экспериментов  
    
#### 1. Подготовка данных    
  
1. Скачайте датасеты с Kaggle:  
   - Titanic: `train.csv`, `test.csv`  
   - House Prices: `train.csv`, `test.csv`  
2. Разместите файлы в директориях (или скорректируйте пути в `config.yaml`):  
   - `./titanic/train.csv`, `./titanic/test.csv`  
   - `./pricing/train.csv`, `./pricing/test.csv`  
  
#### 2. Обучение и инференс (Titanic)  
  
1. В `config.yaml`:  
   - в блоке `task` установите:  
     - `titanic: True`  
     - `house_pricing: False`  
   - включите нужные модели в блоке `mode` (например, `RandomForest: True`, `CatBoost: True`, `XGB: True` и т.д.).  
   - при необходимости скорректируйте гиперпараметры в блоке `params`.  
2. Запустите:  
  
```bash  
python main.py  
```  
  
При `mode.is_train: True` модели обучатся с кросс‑валидацией, чекпоинты и OOF‑предсказания сохранятся в директории `checkpoints`.    
При `mode.is_inference: True` будет выполнен инференс, ансамблирование и сформирован файл сабмишна `titanic/submission.csv`.  
  
#### 3. Обучение и инференс (House Prices)  
  
1. В `config.yaml`:  
   - в блоке `task` установите:  
     - `titanic: False`  
     - `house_pricing: True`  
   - включите нужные модели в блоке `mode` (например, `LinearRegression`, `Lasso`, `Ridge`, `ElasticNet`, `KNN`, `RandomForest`, `XGB`, `LGBM` и т.д.).  
   - в блоке `inference.ensemble` при желании включите стакинг: `is_stacking: True`.  
2. Запустите:  
  
```bash  
python main.py  
```  

Будут обучены регрессионные модели, сохранены OOF‑предсказания и, при включенном стакинге, обучен мета‑модель `LinearRegression`.  
При инференсе будет сформирован файл сабмишна `pricing/submission.csv`.  
  
### Логика обучения и ансамблей  
  
- Для каждой модели выполняется K‑fold (или StratifiedKFold) обучение; модели по фолдам сохраняются в `checkpoints/<ModelName>/`.  
- Для House Prices дополнительно сохраняются OOF‑предсказания (`*_oof.npy`), которые используются как признаки для стакинга.  
- Для Titanic в режиме инференса считается матрица корреляций предсказаний моделей; если между моделями слишком высокая корреляция, это подсвечивается в логах, чтобы избежать «переусиленного» ансамбля.  

  
Они не участвуют напрямую в пайплайне `main.py`, но полезны для понимания данных и обоснования признаков.  
  
### Как адаптировать под другие задачи  
  
Архитектура проекта позволяет относительно просто портировать решение на другие табличные задачи:  
  
- реализовать новый класс датасета;  
- добавить новый флаг задачи в блок `task` конфига и соответствующую ветку в `Solver`;  
- при необходимости добавить новые модели и гиперпараметры в `config.yaml`.
